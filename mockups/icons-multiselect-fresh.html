multi_select, reconsidered

All ten previous tries were stuck on the same metaphor — "more rectangles." But select already is a rectangle. What makes the multi version semantically different isn't the count of shapes; it's the act of picking. Five fresh starts, each with a real conceptual premise:

A · BUBBLE SHEET
The ballot
Four small circles in a row, two filled — like a multiple-choice form. Reads instantly as "pick any of these."
The only icon in the catalog that uses radio-bubble syntax. Distinct from checkbox (singular, square) by the multiplicity and the round shape. Holds up at 14px because the filled vs hollow contrast is high.

B · VENN
The intersection
Two overlapping circles. Pure set theory — membership in multiple sets at once.
Mathematical elegance. The visual lean is unique in the icon set (no other icon uses overlapping curves). Reads as "things that belong to more than one bucket" — which is exactly multi-select.

accent
C · STREAMERS
The handle
A single tag knot at the left with three ribbons fanning out. One anchor, many strands — like a key ring with several keys.
Borrows from the tags icon family (you already have one) but keeps the multi-ness through the fan, not stacking. Asymmetric, has rhythm, doesn't look like another rectangle.

D · OVERFLOW
The and more
A single chip with a small "+2" floating top-right — like a notification badge for "two more behind this one."
Borrows the universally-understood badge pattern (unread mail count, notification bell). The semantic load lives in the +N, not in shape repetition. Most "literally readable" of the five.

E · CONSTELLATION
The web
Four dots connected by thin lines into a Y-shape. Picked items linked into a small cluster.
The only icon in the set that's a graph. Implies relations between selected items rather than a list of them. Reads slightly more abstract — distinctive for a knowledge tool like Lotion specifically.
