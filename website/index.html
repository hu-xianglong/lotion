Data ownership without interface compromise

Your workspace should not disappear into an app.

Lotion keeps the interface approachable and the storage transparent. You work with pages, databases, and views. Underneath, standard files keep your knowledge portable, inspectable, and useful outside Lotion.

A workspace you can understand

Files are the source of truth.

Lotion does not require a database server or proprietary content store. Markdown carries page content, CSV carries records, JSON carries schema and view configuration, and attachments stay in the workspace.

  • Readable with standard tools
  • Friendly to Git diffs and backups
  • Accessible to scripts, CLIs, and LLM workflows
  • Usable even when Lotion is not running

Bring your existing workspace

A Notion export should become usable files, not a dead archive.

Lotion maps Notion HTML and CSV exports into pages, databases, row pages, links, attachments, and source references. Import reports make skipped or ambiguous content visible instead of silently discarding it.

Local by architecture

The UI never gets unrestricted filesystem access.

Lotion separates React rendering, Electron preload APIs, and main-process file services. Workspace paths and durable operations are validated at the desktop boundary, making local storage testable and integrations independent of UI internals.

Open source and early

Run Lotion locally.

Lotion is currently source-first and best suited to developers and early adopters. The repository includes a synthetic demo workspace, regression fixtures, and performance gates.

Quick start 
git clone https://github.com/hu-xianglong/lotion.git
cd lotion
npm install
npm run demo:reset
npm run dev
